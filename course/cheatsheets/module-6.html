Module 6 — Cheat Sheet

Coding & Technical Work with AI

Coding Prompt Templates
Write new code
"Write a [language] function that [what it does]. Input: [input type/format]. Output: [output type/format]. Requirements: [any constraints, edge cases, libraries to use]. Include comments explaining the logic."
Explain code
"Explain this code to me like I'm a junior developer. Walk through what each section does, why it's structured this way, and any non-obvious decisions: [paste code]"
Refactor / improve
"Here's working code: [paste code]. Refactor it to be cleaner and more readable. Preserve all behavior. Explain what you changed and why. Don't add new functionality."
Debug an error
"I'm getting this error: [paste exact error message]. Here's the relevant code: [paste code]. Here's what I've already tried: [attempts]. Diagnose the cause and give me a fix."
Write tests
"Write unit tests for this function in [testing framework]: [paste code]. Cover: happy path, edge cases, and error conditions. Each test should have a descriptive name."
Review code
"Review this code for: (1) bugs, (2) security issues, (3) performance problems, (4) readability. Be specific — point to exact lines. Rate severity as High/Med/Low: [paste code]"
Debugging Checklist — Before Asking AI
1
Paste the exact errorNever paraphrase error messages. Copy them verbatim — the specific wording matters.
2
Include the full stack traceThe line number and call chain are critical context. Don't trim them.
3
Share the relevant code blockNot the whole file — just the function or section involved. Add context on what it's supposed to do.
4
State what you expected vs. what happenedExpected: X. Got: Y. This frames the problem precisely.
5
List what you've already triedPrevents AI from suggesting things you've already ruled out.
6
Name your environmentLanguage version, framework, OS, any relevant dependencies. Bugs are often version-specific.
No-Code Tool Recommendations
Cursor
AI-native code editor. Understands your whole codebase. Best for developers who want AI embedded in their workflow.
Best for: developers + power users
Replit (Ghostwriter)
Browser-based IDE with AI assist. No local setup needed. Great for quick prototypes and learning.
Best for: beginners + fast prototyping
Bubble + AI
Visual no-code builder. Use Claude/ChatGPT to design logic, then implement visually. No code required.
Best for: non-coders building web apps
Bolt.new
Describe an app in plain text, get working code instantly. Full-stack in seconds. Best for prototyping.
Best for: rapid app prototyping
v0 by Vercel
Generate UI components from text descriptions. React/Tailwind output. Instant frontend scaffolding.
Best for: frontend UI generation
Zapier / Make
Visual automation builders. Use AI to design the workflow, then point-and-click to implement.
Best for: workflow automation
Common Mistakes
  • Pasting vague descriptions instead of actual code and error messages.
  • Accepting AI code without reading it. Always understand what it does before running it.
  • Asking AI to fix code without telling it what the code is supposed to do.
  • Not specifying the language version or framework — AI will make assumptions that may be wrong.
This Week's Action Items
  • Use the debugging checklist the next time you hit a bug — before asking AI or Stack Overflow.
  • Pick one script or piece of code you use regularly and ask AI to refactor it for readability.
  • Try Cursor or Bolt.new for your next technical task — even if you don't usually code.
  • Ask AI to review a piece of your code for security and performance issues. Act on at least one finding.
