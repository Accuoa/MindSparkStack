Module 7 — Cheat Sheet

Advanced Automation and Agent Workflows

Key Concepts
  • An agent is an AI that can take actions — not just respond. It calls tools, browses the web, writes files, and makes decisions in a loop.
  • The difference between a chatbot and an agent: a chatbot responds once. An agent reasons, acts, checks results, and continues until the goal is done.
  • Tools are what agents act with — APIs, browser access, code execution, file operations. The tools define what's possible.
  • Agents fail gracefully when each step is deterministic and verifiable. Design for failure — assume steps will break.
  • Start simple: automate one 3-step workflow before building a 10-step agent pipeline.
  • Human-in-the-loop is not a weakness — it's a best practice until you've verified the agent is reliable end-to-end.

The Plan → Tool Call → Evaluate loop repeats until the goal is achieved or a stop condition is hit. Always define a maximum iteration limit.

Automation Tool Comparison
Zapier
No-Code
Visual workflow builder. 6,000+ integrations. Best for connecting apps without code. Limited branching logic.
Make (Integromat)
No-Code
More powerful than Zapier. Complex branching, data transformation. Steeper learning curve but more flexible.
n8n
Low-Code
Open-source, self-hostable. Full control. Write custom code within nodes. Best for technical users wanting no vendor lock-in.
LangChain / LangGraph
Code
Python framework for building agent pipelines. Full control over logic and memory. Requires development skills.
Claude Agents SDK
Code
Anthropic's native SDK for multi-agent workflows. Best for Claude-native pipelines with tool use and orchestration.
Relevance AI
Low-Code
Build AI agents visually. Chain LLM calls with tools. Good middle ground between no-code and custom development.
First Agent Project Checklist
Define the triggerWhat starts the workflow? Schedule, webhook, manual run, or event?
Map the steps (3 max to start)List what the agent must do. Keep it linear before adding branching.
Identify the tools neededWhich APIs, apps, or data sources does each step require?
Define success criteriaHow will you know the agent completed the task correctly?
Test with manual inputs firstRun each step by hand before automating. Verify outputs at each stage.
Add error handlingWhat happens if step 2 fails? Define fallbacks before going live.
Monitor the first 10 runsWatch every run and log unexpected behavior before running unsupervised.
Quick-Reference Prompts
Design an agent workflow
"I want to automate this workflow: [describe it]. Break it into discrete steps. For each step, identify: what triggers it, what data it needs, what tool it uses, and how to verify it succeeded."
Identify automation opportunities
"Here's how I currently do [task] manually: [describe steps]. Which parts are repetitive and rule-based enough to automate? Rank them by impact and implementation difficulty."
Debug an agent failure
"My agent failed at step [N]. Here's the input it received: [data]. Here's what it was supposed to do: [expected]. Here's what happened: [actual]. What likely went wrong and how do I fix it?"
Common Mistakes
  • Building 10-step agents before mastering 2-step automations.
  • No error handling — one failed API call breaks the whole chain.
  • Automating a workflow you haven't done manually enough times to know all the edge cases.
  • Skipping the monitoring phase — agents behave differently at scale.
