Module 4 — Cheat Sheet

Building Your Personal AI Stack

Tool Selection Decision Tree
Is this a writing/thinking/research task?
→
Start with Claude or ChatGPT (Plus/Pro). These are your workhorses. Pick the one you've invested time in learning.
Do you need current web information?
→
Use Perplexity Pro or enable web browsing in Claude/ChatGPT. Training data has a cutoff — don't trust it for recent events.
Is it a code or technical task?
→
GitHub Copilot or Cursor for in-editor assistance. ChatGPT/Claude for architecture decisions and explanation.
Is it image/video/audio generation?
→
Images: Midjourney (quality) or DALL-E (convenience). Video: Runway/Sora. Audio/Voice: ElevenLabs.
Can a general tool handle it?
→
Test with Claude/ChatGPT first. Only pay for a specialist tool if the general tool clearly falls short after a real attempt.

Quick-Reference Prompts

"I'm evaluating [tool] for [use case]. What are the key questions I should test it on to determine if it's better than [current tool]?"
Audit your stack
"I pay for [list tools]. My top 3 use cases are [list]. Which tools are redundant and which are essential? What am I missing?"
Common Mistakes
  • Paying for specialist tools before mastering the generalist ones. Go deep on one before adding more.
  • Building a stack based on hype rather than your actual workflow and use cases.
  • Tool-switching constantly. Give each tool 2 weeks before judging it.
  • Ignoring the free tier. Most tasks don't need Pro — don't upgrade before you hit the ceiling.
