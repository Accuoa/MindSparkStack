Module 3 — Cheat Sheet

The 5 Workflow Patterns

The 5 Patterns at a Glance
1
Generate — Single Shot
One prompt, one output. Use when the task is self-contained and well-defined. Draft, summarize, translate, reformat.
Use when: You need a first draft fast, the requirements are clear, and iteration is low-stakes.
Pattern 1
2
Refine — Iterative Loop
Generate, review, prompt again. Use for anything that needs polish — writing, code, strategies, plans.
Use when: Quality matters more than speed, and you need the output to meet a high bar.
Pattern 2
3
Decompose — Break it Down
Split complex tasks into sub-tasks. Handle each part separately, then combine. Great for research, long-form writing, complex projects.
Use when: The task is too big to handle in one shot or requires different expertise per section.
Pattern 3
4
Critique — Evaluate & Improve
Give AI existing work and ask it to critique, stress-test, or find gaps. Use on your own drafts or to review decisions.
Use when: You have something and need honest feedback, not just validation.
Pattern 4
5
Automate — Chain & Repeat
Build a repeatable sequence of prompts that runs on any input. Templates, pipelines, recurring workflows. Set it up once, use it forever.
Use when: You do the same type of task repeatedly and want to standardize quality and speed.
Pattern 5
Quick-Start Prompt for Each Pattern
Pattern 1 — Generate
"Write a [format] about [topic] for [audience]. Tone: [tone]. Length: [length]. Include: [specific elements]."
Pattern 2 — Refine
"Here's my draft: [paste draft]. Improve it for [specific goal]. Keep [what to preserve]. Change [what to fix]. Show me the revised version only."
Pattern 3 — Decompose
"I need to [big task]. Before starting, break this into the key sub-tasks I should address. Then we'll tackle each one separately."
Pattern 4 — Critique
"Here's [my plan/draft/decision]: [paste it]. Act as a skeptical expert. What are the 3 biggest weaknesses, risks, or gaps? Be direct — no softening."
Pattern 5 — Automate
"I need a reusable template for [recurring task]. Create a prompt template I can fill in each time, with [INPUT] placeholders for the parts that change."
Common Mistakes
  • Using Pattern 1 for everything — complex tasks need decomposition or iteration, not one-shots.
  • Skipping Pattern 4 — most people generate but never critique. The best outputs come from generating then stress-testing.
  • Building automations (Pattern 5) before you've refined the prompt manually first.
  • Mixing patterns in one messy prompt instead of running them as separate, clean steps.
This Week's Action Items
  • Identify which pattern you overuse. (Most people default to Pattern 1 for everything.)
  • Run Pattern 4 on your best piece of work this week — give it to AI for a harsh critique.
  • Pick one recurring task and build a Pattern 5 template for it. Use it 3 times to refine it.
  • Try Pattern 3 on your next complex project — decompose before diving in.
